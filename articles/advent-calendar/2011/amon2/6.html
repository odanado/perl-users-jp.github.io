さて、今日は Amon2 と bootstrap についてかこうかとおもいます。

Why bootstrap?

Amon2 では setup script を非常に重視していることは有名ですが、これはつまり、「なんかあたらしいサイトをつくりたいなー」とおもったときに、そのサイトの中心となるコード以外にとらわれたくないということです。中心的関心事以外のわずらわしいことからの開放、それが重要です。

そこで css framework とかいうものが重要となってきます。

なにしろ個人的に、programmer 手でいちからかいた html + css みたいなやつのツールとかをつかわされると非常に残念な気分になるときがあって、それが嫌なんです。そもそも font-face をあててなかったりとか、なんか window のはじにくっついちゃってみにくかったり、表示要素がおおくなると sidebar が下におっこってたりとかね。

たとえば、css がよくわかってない programmer がかいた css とか、2011 年にもなってサイドバーが下におっこったりすることがあったりして残念な気持になるわけですが、いわゆる gird system 的なやつつかっとけば、まあそういう事故はなくなりますすよ。

まあ bootstrap css つかっとけば、最低限、これがここに表示されてほしい、というのは誰でもかけるよになるので、まあ、赤点にはならない見た目が達成されます。

見た目がおなじサイトばっかにならない?

まあひとつそういう問題点はあるんですが、なんかもうプログラマがセンスのかけらもかんじられないデザインセンス()を発揮するよりはいいんじゃないかなとおもいます。

そしてまあ bootstrap css っていうのは bootstrap css っていうのはもともとが *bootstrap* なんで、べつにこれ自体をいじりゃいいんですね。bootstrap css は less で構成されているんで、構成要素が構造化されていて、わりといじりやすいですよ。

あとまあ、とりあえず bgcolor を指定して top bar の色かえるとだいたい雰囲気かわるので、そこだけがんばってみるといいかとおもいます。そのへん次のバージョンあたりでいじりやすくするっていってましたけどどうなることやら。

もっとシンプルでいいんだけど

とおもう方は、まあいらない要素をけずりつつ、どんどんいじったらいいんじゃないすかね。

まとめ

bootstrap は、いじりたおせば結構いい。
